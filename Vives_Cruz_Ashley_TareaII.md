**Tecnologías:** React y Firebase  

 1. Descripción del proyecto

Tero Match es una app sencilla para recomendar perfumes.

La idea es que el usuario pueda entrar, ver perfumes, contestar unas preguntas y recibir una recomendación según sus gustos. También puede guardar perfumes favoritos para revisarlos después.

El proyecto se haría con React para la parte visual y Firebase para guardar datos.

5. Criterios de aceptación

HU01 - Registro

- El usuario puede registrarse.
- El usuario puede iniciar sesión.
- Si deja campos vacíos, la app muestra un mensaje.

HU02 - Catálogo

- Se ven los perfumes guardados en Firebase.
- Cada perfume muestra nombre, precio y descripción.
- La pantalla se puede ver en celular y computadora.

HU03 - Recomendación

- El usuario puede responder las preguntas.
- La app muestra una recomendación final.
- La recomendación se entiende de forma clara.

HU04 - Favoritos

- El usuario puede guardar un perfume favorito.
- El usuario puede ver sus favoritos.
- El usuario puede quitar un favorito si quiere.


6. Justificación de la prioridad

Primero puse las historias de registro, catálogo y preguntas porque son la base de Tero Match.

Sin esas partes, la app no podría recomendar perfumes. Los favoritos quedan dentro del sprint porque ayudan al usuario a guardar opciones.

El filtro por precio queda para después, porque es útil, pero no es obligatorio para la primera versión.
